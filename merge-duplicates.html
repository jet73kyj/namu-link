📋 병합 규칙

동일 (이름 + 생년월일) 그룹에 대해 자동 판정:
1) 유지할 레코드 선정 우선순위:
  ① status='종결' 인 레코드 우선 (종결 이력 보존)
  ② phase='active' + 치료 개수 많은 순
  ③ managerId 있는 레코드
  ④ 가장 오래된 createdAt
2) 병합 필드:
  • therapies: 유지 레코드 + 나머지 유일 (support+cycle+timeSlot) 로 dedup 후 union
  • managerId · guardian · phone: 유지 레코드가 비어있으면 다른 값으로 채움
  • status: 어느 하나라도 '종결' 이면 '종결' 로 병합
  • intakeArea/Pref/Program/Note: 유지 레코드가 비어있으면 채움
3) 연동 데이터 이관:
  • payments 의 childId → 유지 레코드 id 로 변경
  • assessment_events 의 childId·id 접두어 → 유지 id 로
  • child_docs_/timeline_/parent_memos_ 키 재작성
